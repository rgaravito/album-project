# Specs Folder

Product Agent outputs feature specifications here.

## Naming Convention

`<feature-name>.md`

Examples:
- `user-authentication.md`
- `card-marketplace.md`
- `exchange-system.md`

## Spec Lifecycle

1. **Created**: Product Agent creates spec from ROADMAP
2. **Reviewed**: Team reviews spec completeness
3. **Reference**: Backend/Frontend Agents use as requirements
4. **Closed**: Marked complete when feature ships
