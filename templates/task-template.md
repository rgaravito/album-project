# Task Template

Use this template to break down specs into actionable development tasks.

## Title
```
# Task: <Task Name>
```

## Sections

### Description
2-3 sentences describing the work to be done.

### Acceptance Criteria
Checkbox list of deliverables.

- [ ] Item 1
- [ ] Item 2
- [ ] Item 3

### Implementation Details

Specific guidance for developers:
- Files to create/modify
- Code examples if helpful
- Data models
- API contracts
- UI/UX specifics

### Testing

How to verify the task is complete.

### Assigned To

Which agent(s) should work on this.

Example:
- Backend Agent
- Frontend Agent
- Testing Agent

### Notes

Any additional context:
- Dependencies on other tasks
- Tools/libraries to use
- Performance considerations
- Security considerations

---

## Task Naming

Format: `NN-<feature-name>.md`

Examples:
- `01-project-setup.md`
- `05-card-catalog-frontend.md`
- `10-user-profile.md`

Number sequentially in order of execution (Phase 1, then Phase 2, etc.)

## Good Task Properties

✅ Completable in 2-4 hours
✅ Has clear success criteria
✅ Independent or clearly depends on X task
✅ Includes implementation guidance
✅ Specifies who should work on it

❌ Too vague
❌ Too large (takes > 8 hours)
❌ Missing technical details
❌ No clear testing method
